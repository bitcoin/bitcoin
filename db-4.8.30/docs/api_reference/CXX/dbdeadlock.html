DbDeadlockException

#include <db_cxx.h>

class DbDeadlockException : public DbException { ... };

This information describes the DbDeadlockException class and how it is used by the various Berkeley DB classes.

A DbDeadlockException is thrown when multiple threads competing for a lock are deadlocked, when a lock request has timed out (and DB_TIME_NOTGRANTED has not been set in the environment), or when a lock request would need to block and the transaction has been configured to not wait for locks. One of the threads' transactions is selected for termination, and a DbDeadlockException is thrown to that thread.

The DbException errno value is set to DB_LOCK_DEADLOCK.
